Create `src/utils/fuelPrices.ts` — a standalone utility module that owns the full fetch-cache-fallback lifecycle.

**Exported interface:**
```typescript
export interface LiveFuelPrices {
  petrol: number    // dollars per litre
  diesel: number    // dollars per litre
  isEstimated: boolean  // true when using fallback, false when from API
}
```

**Exported constant — FALLBACK_PRICES (per D-09, values from RESEARCH.md):**
```typescript
export const FALLBACK_PRICES: LiveFuelPrices = {
  petrol: 2.30,
  diesel: 2.85,
  isEstimated: true,
}
```

**Module-level constant:**
```typescript
const CACHE_MAX_AGE_MS = 24 * 60 * 60 * 1000  // 24 hours (per D-13)
```

**Exported function — loadFuelPriceCache(uid: string): Promise&lt;LiveFuelPrices | null&gt;**
- Import `doc, getDoc` from `firebase/firestore` and `db` from `../firebase`
- Call `getDoc(doc(db, 'users', uid))`
- If `!snap.exists()` return `null`
- Read `snap.data()?.fuelPriceCache`
- If `!cache || !cache.cachedAt` return `null`
- Compute `ageMs = Date.now() - cache.cachedAt.toMillis()` (uses Timestamp.toMillis per Pitfall 6 from RESEARCH.md)
- If `ageMs > CACHE_MAX_AGE_MS` return `null` (stale)
- Return `{ petrol: cache.petrol, diesel: cache.diesel, isEstimated: false }`
- Wrap entire function body in try/catch, catch returns `null`

**Exported function — saveFuelPriceCache(uid: string, petrol: number, diesel: number): Promise&lt;void&gt;**
- Import `updateDoc, Timestamp` from `firebase/firestore`
- Call `updateDoc(doc(db, 'users', uid), { fuelPriceCache: { petrol, diesel, cachedAt: Timestamp.now() } })`
- This mirrors the `saveDriveCache()` pattern — merges into existing user doc

**Exported function — fetchFuelPrices(suburb: string): Promise&lt;LiveFuelPrices&gt;**
- Read API key: `const apiKey = import.meta.env.VITE_FUELPRICE_API_KEY as string | undefined`
- If `!apiKey`, log `console.warn('VITE_FUELPRICE_API_KEY not set — using fallback fuel prices')` and return `FALLBACK_PRICES`
- Resolve city from suburb using internal `resolveCity(suburb)` helper (always returns `'Melbourne'` per D-02 — all users are Victorian)
- Make two parallel `fetch` calls via `Promise.all`:
  - Petrol: `https://fuelprice.io/api/?key=${apiKey}&action=get_city_average&fuel_type=unleaded&city=${encodeURIComponent(city)}`
  - Diesel: `https://fuelprice.io/api/?key=${apiKey}&action=get_city_average&fuel_type=diesel&city=${encodeURIComponent(city)}`
- **IMPORTANT NOTE in code comment:** The endpoint format above is from fuelprice.io legacy docs (RESEARCH.md assumption A1). The current v1 API may use `https://api.fuelprice.io/v1/cities/{CITY_ID}/average` with a Bearer token. The implementer MUST verify the correct endpoint after obtaining an API key. If this endpoint format is wrong, the catch block returns FALLBACK_PRICES gracefully.
- If either response is not `.ok`, return `FALLBACK_PRICES`
- Parse both responses as JSON. Expected shape: `{ average?: number }` (cents per litre)
- If either `average` is falsy, return `FALLBACK_PRICES`
- Convert cents to dollars: `petrol / 100, diesel / 100`
- Return `{ petrol: petrol / 100, diesel: diesel / 100, isEstimated: false }`
- Wrap entire function body in try/catch; catch logs `console.error('fetchFuelPrices failed, using fallback')` and returns `FALLBACK_PRICES`

**Internal helper — resolveCity(suburb: string): string**
- Always returns `'Melbourne'` — per D-02, all users are Victorian and regional variance is not worth the complexity
- Not exported

**Import organization (following project conventions):**

