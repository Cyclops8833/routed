**Change 2 (D-08) — lines 204-207:** Delete the entire global registration block:
```typescript
  // Register global plan-trip callback so the popup button can reach React
  if (options?.onPlanTrip) {
    ;(window as Window & { __routedPlanTrip?: (id: string) => void }).__routedPlanTrip = options.onPlanTrip
  }
```
Remove these 4 lines entirely.

**Change 3 (D-06) — after `popup.addTo(map)` on line 228:** Add a `popup.on('open')` listener. The `destId` variable is `String(props.id)` already in scope from line 217. After line 228 (`.addTo(map)`), add:

```typescript
    popup.on('open', () => {
      const btn = popup?.getElement()?.querySelector('button')
      if (btn && options?.onPlanTrip) {
        btn.addEventListener('click', () => {
          options.onPlanTrip!(String(props.id))
        })
      }
    })
```

Note: Use `String(props.id)` directly in the click handler (not the outer `destId` variable from `buildPopupHtml` call which is the same value but clearer to reference `props.id` since we are inside the click handler scope where `props` is available). Actually, looking at the code, `destId` is not a variable in the click handler scope -- it was only passed to `buildPopupHtml`. Use `String(props.id)` which IS in scope from the enclosing `map.on('click')` callback.
