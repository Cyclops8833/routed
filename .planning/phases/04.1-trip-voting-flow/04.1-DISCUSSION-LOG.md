## Problem Statement

User reported: unable to test Phase 4 fuel price integration because they can't move a trip to confirmed/voting state. Specific failure: click "Continue to Voting (2 destinations)" in TripSheet → returns to Trips tab → nothing there.

## Codebase Analysis Findings

- "Open voting" button **does exist** in TripDetail (lines 944-996), behind `status === 'proposed'` + `isCreator` — feature is implemented, not missing
- TripSheet submit handler (`handleContinueToVoting`) writes to Firestore with `status: 'proposed'`, then calls `onClose()` + `navigate('/trips')`
- `DirectTripSheet` uses the correct pattern: navigates to `/trips/${ref.id}` after creation
- TripSheet error handling: `catch (err) { console.error(...) }` — silent failure to user
- Trips.tsx query: `where('attendees', 'array-contains', uid)` — if `attendees: []`, trip is invisible
- Crew loading: if `getDocs(collection(db, 'users'))` fails, `selectedAttendees` stays empty and `attendees` written as `[]`
