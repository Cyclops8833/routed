# Plan 05-01: CrewContext — eliminate 7 duplicate Firestore reads

**Phase:** 05 — Performance foundation
**Requirement IDs:** PERF-CREW-CTX
**Executor:** gsd-executor

## Objective

Create a shared `CrewContext` that maintains a single `onSnapshot(collection(db, 'users'))` subscription and exposes `allUsers: UserProfile[]` to any component that needs it. Mount the provider in `App.tsx` alongside `NotificationProvider`. Migrate all 7 callsites that currently issue independent `getDocs(collection(db, 'users'))` calls to consume the context instead, eliminating duplicate Firestore reads on every page load.

## Tasks

### Task 1: Create `src/contexts/CrewContext.tsx`

**File:** `src/contexts/CrewContext.tsx`

Model the file directly on `NotificationContext.tsx`. The context needs no `onAuthStateChanged` guard because the `users` collection is readable by any authenticated user; but mirror the guard from `NotificationContext` (set uid via `onAuthStateChanged`, subscribe only when uid is non-null) to avoid an unauthenticated read:

```tsx
import { createContext, useContext, useEffect, useState, type ReactNode } from 'react'
import { collection, onSnapshot } from 'firebase/firestore'
import { onAuthStateChanged } from 'firebase/auth'
import { db, auth } from '../firebase'
import type { UserProfile } from '../types'

interface CrewContextValue {
  allUsers: UserProfile[]
}

const CrewContext = createContext<CrewContextValue>({ allUsers: [] })

export function useCrewContext(): CrewContextValue {
  return useContext(CrewContext)
}

export function CrewProvider({ children }: { children: ReactNode }) {
  const [allUsers, setAllUsers] = useState<UserProfile[]>([])
  const [uid, setUid] = useState<string | null>(null)

  useEffect(() => {
    return onAuthStateChanged(auth, (user) => {
      setUid(user?.uid ?? null)
    })
  }, [])

  useEffect(() => {
    if (!uid) {
      setAllUsers([])
      return
    }
    const unsub = onSnapshot(collection(db, 'users'), (snap) => {
      setAllUsers(snap.docs.map((d) => d.data() as UserProfile))
    })
    return unsub
  }, [uid])

  return (
    <CrewContext.Provider value={{ allUsers }}>
      {children}
    </CrewContext.Provider>
  )
}
```

**Verify:** File exists at `src/contexts/CrewContext.tsx` and TypeScript compiles (`npx tsc --noEmit`).

## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| `onSnapshot` fires before auth is ready, exposing empty user list to UI | Low | Provider guards subscription behind `uid !== null` via `onAuthStateChanged` — same pattern as NotificationContext |
| Context `allUsers` contains stale data if Firestore offline | Low | `onSnapshot` uses local Firestore cache when offline; data may be stale but not missing |
| `crewMembersRef` drifting out of sync with context in Mapbox closures | Med | Explicit `useEffect(() => { crewMembersRef.current = allUsers }, [allUsers])` keeps ref current on every context update (D-05) |
| TripDetail `useMemo` attendee filter including wrong users if `trip.attendees` is stale | Low | Memo dep is `trip?.attendees.join(',')` — re-derives whenever attendee list changes |
</threat_model>
