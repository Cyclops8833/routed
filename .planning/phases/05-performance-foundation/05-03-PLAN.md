## Objective

Prevent unnecessary re-renders of two expensive card components. Extract `SpotlightCard` from `Map.tsx` into its own file and wrap it with `React.memo`. Wrap `DestinationCard` with `React.memo`. Add this alongside the other component imports at the top of the file.

**2c — Wrap `handleSpotlightTap` with `useCallback` (D-12):**

Current definition (near line 649 after Plan 05-01 changes):
```tsx
const handleSpotlightTap = (dest: Destination) => {
  const map = mapRef.current
  if (!map) return
  map.flyTo({ center: [dest.lng, dest.lat], zoom: 10 })
}
```

Replace with:
```tsx
const handleSpotlightTap = useCallback((dest: Destination) => {
  const map = mapRef.current
  if (!map) return
  map.flyTo({ center: [dest.lng, dest.lat], zoom: 10 })
}, [])
```

Dependency array is empty — the function only reads `mapRef.current` (a ref, not state) so no reactive deps are needed. This produces a stable function reference across re-renders, allowing `React.memo` on `SpotlightCard` to skip re-renders.

Find the JSX where `SpotlightCard` is rendered in the return statement (it renders inside a horizontal scroll container for spotlight destinations). Confirm the props passed match the new file's interface: `dest`, `driveCache`, `userHomeLocation`, `onTap`. No JSX changes should be needed — the extraction is transparent.
